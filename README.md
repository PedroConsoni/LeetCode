## Big O Notation — Complexity Analysis

**Big O Notation** measures the **growth of time or space** an algorithm consumes as the input size (`n`) increases.  
It is essential for evaluating the **efficiency** of LeetCode solutions.

### Time Complexity

| Notation | Type | Example Algorithm |
|----------|------|-------------------|
| O(1) | Constant | Direct array access |
| O(log n) | Logarithmic | Binary search |
| O(n) | Linear | Iterating through an array |
| O(n log n) | Linearithmic | Merge Sort, Quick Sort |
| O(n²) | Quadratic | Bubble Sort |
| O(2ⁿ) | Exponential | Recursive Fibonacci |
| O(n!) | Factorial | Permutation generation |

### Space Complexity

| Notation | Type | Example |
|----------|------|---------|
| O(1) | Constant | Fixed-size variables |
| O(n) | Linear | Data structures proportional to input size |
| O(n²) | Quadratic | Two-dimensional matrices |
